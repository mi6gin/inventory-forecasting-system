# Inventory Forecasting & Procurement System (LSTM-based)

Профессиональная система прогнозирования спроса и автоматизации закупок, построенная на базе глубокого обучения (LSTM) с учетом экзогенных факторов.

## Архитектурные принципы

Система реализует пайплайн **End-to-End**: от генерации синтетических данных со сложными паттернами до формирования юридически значимого (PDF) плана закупок.

### Ключевые особенности (v2.0 Refactored):
*   **Deep Learning Core:** Архитектура LSTM + MLP для одновременной обработки временных рядов (лаги продаж) и статических/экзогенных признаков (категории, праздники, промо-акции).
*   **Multi-Quantile Loss (Pinball Loss):** Модель предсказывает не только среднее значение, но и квантили (0.1, 0.5, 0.9). Это позволяет формировать "страховой запас" (Safety Stock) на основе 90-го квантиля, минимизируя риск Out-of-Stock.
*   **Авторегрессионная симуляция:** В отличие от простых моделей, данная система строит прогноз итеративно день за днем, что позволяет учитывать динамическое истощение остатков при планировании.
*   **Feature Engineering:** Включает тригонометрическое кодирование времени (sin/cos дня и месяца) для захвата сезонности без раздувания размерности признаков.

## Технические исправления (Audit Fixes)
1.  **Target Consistency:** Исправлена логика прогнозирования — модель предсказывает шаг $t+1$, устраняя математическую ошибку деления агрегированного прогноза на константу.
2.  **Scale Partitioning:** Внедрено раздельное масштабирование признаков. `RobustScaler` применяется только к непрерывным данным, предотвращая порчу бинарных флагов.
3.  **Data Integrity:** Устранена проблема `numpy.object_` через явное приведение типов к `float32` перед подачей в вычислительный граф Torch.
4.  **IO Performance:** Оптимизирована работа с БД через батчевые вставки (`executemany`).

## Структура проекта
*   `main.py` — Orchestrator пайплайна.
*   `data_generator.py` — Генератор синтетической истории продаж с учетом праздников, акций и дефицита.
*   `model_trainer.py` — Обучение LSTM, масштабирование признаков и валидация.
*   `losses.py` — Кастомные функции потерь (MultiQuantileLoss).
*   `recommender.py` — Инференс, симуляция складских остатков и генерация PDF-отчетов.

## Запуск
1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите полный цикл:
   ```bash
   python3 main.py
   ```

## Выходные данные
*   `inventory_forecast.db` — Локальное хранилище данных.
*   `demand_model.pth` — Веса обученной нейросети.
*   `demand_scaler.pkl` — Параметры масштабирования признаков.
*   `reports/` — Итоговые планы закупок в формате PDF с графиками прогнозируемого обнуления стоков.
