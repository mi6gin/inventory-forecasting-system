# Inventory Forecasting & Procurement System (v3.0 Scientific Edition)

Профессиональная система прогнозирования спроса и автоматизации закупок, построенная на базе глубокого обучения (LSTM/Transformer) с учетом экзогенных факторов и квантильной регрессии.

## Архитектурные принципы

Система реализует пайплайн **End-to-End**: от генерации синтетических данных со сложными паттернами до формирования научно-обоснованного (PDF) плана закупок с визуализацией рисков.

### Ключевые особенности (v3.0):
*   **Hybrid Deep Learning:** Поддержка LSTM, GRU и оптимизированного Transformer (с механизмом Global Average Pooling) для захвата долгосрочных зависимостей.
*   **Multi-Quantile Risk Management:** Модель предсказывает 10%, 50% и 90% квантили. Это позволяет строить "коридор неопределенности" и рассчитывать Safety Stock на основе волатильности прогноза, а не констант.
*   **Exogenous Factors:** Учет календаря праздников, промо-акций и дней недели при авторегрессионном прогнозировании (Inference Feature Blindness Fix).
*   **Scientific Validation:** Сравнение нейросетевых моделей с сезонным baseline (SARIMA 1,1,1x1,1,1,7). Расчет метрик WAPE, MAE, RMSE и MAPE.
*   **Individual Metrics:** Оценка точности (WAPE) для каждого товара индивидуально на основе волатильности его спроса.
*   **Visual Analytics:** Автоматическая генерация графиков прогноза с Shadow Area (интервал риска) для топ-5 критических позиций в PDF-отчете.

## Технические исправления (Audit v3.0)
1.  **Feature Blindness Fix:** Исправлена критическая ошибка инференса — теперь модель "видит" будущие праздники и промо-акции при генерации прогноза на 30 дней.
2.  **Transformer Optimization:** Внедрен Global Average Pooling вместо использования только последнего токена, что расширило рецептивное поле модели.
3.  **DRY Refactoring:** Весь код нейросетевых моделей унифицирован через базовый класс `BaseForecaster`.
4.  **Baseline Upgrade:** ARIMA (5,1,0) заменена на SARIMA с недельным периодом (7) для корректного научного сравнения.

## Структура проекта
*   `main.py` — Orchestrator пайплайна.
*   `data_generator.py` — Генератор синтетической истории продаж с учетом праздников, акций и дефицита.
*   `model_trainer.py` — Обучение моделей, кросс-валидация (TimeSeriesSplit) и расчет метрик.
*   `losses.py` — Кастомные функции потерь (MultiQuantileLoss).
*   `recommender.py` — Инференс, генерация графиков и формирование научного PDF-отчета.

## Запуск
1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите полный цикл:
   ```bash
   python3 main.py
   ```

## Выходные данные
*   `inventory_forecast.db` — База данных SQLite.
*   `demand_model.pth` — Веса лучшей модели.
*   `charts/` — Визуализация прогнозов спроса (PNG).
*   `reports/` — Научно-обоснованные планы закупок (PDF) с методологией и аналитикой рисков.
