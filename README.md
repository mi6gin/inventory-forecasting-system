# Inventory Forecasting System (LSTM + Quantile Regression)

Интеллектуальная система прогнозирования товарных запасов на базе глубокого обучения. Проект использует рекуррентную нейросеть (LSTM) для анализа временных рядов и многоквантильную регрессию для оценки рисков дефицита.

## 🚀 Основные возможности

*   **LSTM-архитектура:** Нейросеть учитывает последовательность продаж и долгосрочные зависимости.
*   **Экзогенные признаки:** Модель обучается на внешних факторах, таких как праздники, рекламные акции и сезонность.
*   **Учет дефицита (Out-of-Stock):** Алгоритм понимает разницу между отсутствием спроса и отсутствием товара на полке.
*   **Многоквантильный прогноз:** Система выдает не одно число, а три сценария:
    *   `Q10` (Оптимистичный — низкий спрос)
    *   `Q50` (Медианный — наиболее вероятный)
    *   `Q90` (Пессимистичный — высокий спрос, используется для безопасного запаса)
*   **Автоматический план закупок:** Генерация PDF-отчета с цветовой индикацией приоритетов (Критично / Планово / Норма).

## 📊 Структура базы данных (SQLite)

Для работы системы требуется база `inventory_forecast.db`. Ниже приведена схема таблиц:

### 1. Таблица `products` (Справочник товаров)
| Колонна | Тип | Описание |
| :--- | :--- | :--- |
| `id` | INTEGER | Уникальный ID товара (PK) |
| `name` | TEXT | Наименование товара |
| `category` | TEXT | Категория (для One-Hot кодирования) |
| `lead_time` | INTEGER | Срок поставки товара в днях |
| `unit_price`| REAL | Цена за единицу |

### 2. Таблица `sales_history` (История продаж и факторы)
| Колонна | Тип | Описание |
| :--- | :--- | :--- |
| `product_id` | INTEGER | ID товара (FK) |
| `sale_date` | DATE | Дата продажи |
| `quantity_sold`| INTEGER | Проданное количество |
| `in_stock` | INTEGER | Был ли товар в наличии (1 - да, 0 - нет) |
| `is_holiday` | INTEGER | Является ли день праздничным (1/0) |
| `is_promo` | INTEGER | Проходила ли акция (1/0) |

### 3. Таблица `warehouse_stock` (Текущие остатки)
| Колонна | Тип | Описание |
| :--- | :--- | :--- |
| `product_id` | INTEGER | ID товара (FK) |
| `current_quantity`| INTEGER | Текущее количество на складе |

## 🛠 Технологический стек

*   **Core:** Python 3.9+
*   **ML Framework:** PyTorch (LSTM, MultiQuantileLoss)
*   **Data Analysis:** Pandas, NumPy, Scikit-learn
*   **Database:** SQLite3
*   **Reporting:** ReportLab (PDF), Matplotlib (Графики)

## 🏃‍♂️ Запуск проекта

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите основной цикл:
   ```bash
   python main.py
   ```

После завершения:
*   В корне появится PDF-файл `Action_Plan_YYYYMMDD.pdf`.
*   В папке `charts/` будут сохранены графики прогноза остатков для каждого товара.

## 📈 Логика цветовой индикации в отчете

*   🔴 **КРИТИЧНО:** Остаток обнулится раньше, чем приедет новая поставка. Требуется срочное решение.
*   🟡 **ПЛАНОВАЯ:** Время для заказа подошло (согласно Lead Time).
*   ⚪ **НОРМА:** Запасов достаточно на ближайшие 30+ дней.
